# ADR-0001: Body-State Taxonomy and Notation
**Document ID**: ADR-0001  
**Version**: 1.0  
**Last Updated**: 2024-12-19  
**Status**: Active

## Context

The USE Engine requires a precise, measurable representation of player body states to enable realistic tackle success modeling, fatigue accumulation, and injury risk assessment. The system must capture complex physical interactions including leverage, positioning, and movement dynamics in a way that supports both real-time simulation and analytical validation.

Key forces at play:
- **Precision vs. Performance**: Detailed body state tracking must not compromise simulation performance
- **Measurability vs. Intuition**: States must be quantifiable for validation while remaining intuitive for designers
- **Completeness vs. Complexity**: Comprehensive coverage of football-specific movements vs. manageable state space
- **Real-time vs. Analytical**: States must support both real-time game simulation and post-game analysis

The existing research emphasizes the importance of realistic player attributes and dynamic outcomes【message_idx†source】, requiring a body state system that can accurately model the physical nuances that determine tackle success and player safety.

## Decision

We will implement a normalized body state taxonomy using standardized units and discrete bins for continuous variables. The system will use:

1. **Normalized angles** (0-360° for yaw, -90° to +90° for pitch/roll) with 5° bins for computational efficiency
2. **Velocity bins** (0-15 m/s in 0.5 m/s increments) to balance precision with performance
3. **Discrete state categories** for locomotion, orientation, contact-prep, aerial/grounded, and ball control
4. **Composite indices** (e.g., Leverage Index) that combine multiple normalized factors

Each body state will have a unique ID (format: `BOD-XXX-###`) and explicit entry/exit conditions expressed as measurable thresholds.

## Alternatives

**Alternative 1: Continuous Floating-Point Representation**
- Description: Use precise floating-point values for all angles, velocities, and positions
- Pros: Maximum precision, smooth transitions, no quantization artifacts
- Cons: Performance overhead, complex validation, difficult to debug and analyze
- Why Rejected: Performance requirements and the need for discrete state validation outweigh precision benefits

**Alternative 2: Binary State Flags**
- Description: Use simple boolean flags for each possible body state
- Pros: Simple implementation, fast computation, easy validation
- Cons: Cannot represent continuous variables, lacks precision for realistic modeling
- Why Rejected: Insufficient precision for accurate tackle success and injury risk modeling

**Alternative 3: Hierarchical State Machine**
- Description: Use nested state machines with inheritance and composition
- Pros: Rich representation, supports complex relationships, extensible
- Cons: Complex implementation, difficult to validate, performance overhead
- Why Rejected: Overly complex for the current requirements and team capabilities

**Alternative 4: Machine Learning-Based Classification**
- Description: Use ML models to classify body states from raw sensor data
- Pros: Potentially more accurate, can learn complex patterns
- Cons: Requires training data, black-box behavior, difficult to validate and debug
- Why Rejected: Lack of training data and the need for deterministic, explainable behavior

## Consequences

**Positive Consequences:**
- **Performance**: Discrete bins enable fast lookup and computation
- **Validation**: Measurable thresholds support comprehensive testing
- **Debugging**: Clear state boundaries make issues easier to identify
- **Analysis**: Discrete states enable statistical analysis and pattern recognition
- **Consistency**: Normalized values ensure consistent behavior across different scenarios

**Negative Consequences:**
- **Quantization**: Discrete bins may introduce artifacts at boundary conditions
- **Precision Loss**: 5° angle bins may miss subtle but important variations
- **State Explosion**: Large number of possible state combinations requires careful management
- **Complexity**: Composite indices add computational overhead

**Neutral Consequences:**
- **Learning Curve**: Team must learn the taxonomy and notation system
- **Maintenance**: State definitions may need updates as requirements evolve
- **Documentation**: Comprehensive documentation required for all state definitions
